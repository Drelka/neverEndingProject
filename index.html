let, const, var

variables

Variables are containers that store your data. You should declare them before you use them.
Const creates a variable whose reference cannot be reassigned, while let creates a variable that can be reassigned.



let month; // declaration only
month = "January"; // assignment
month = "February"; // re-assignment

const name = "Joe" // declaration and assignment
let age = 22;

name = "Jamie"; // can't reassign value for const, throwing error
age = 33; // new value is valid



There's also a third kind of a variable - var - but it's been depreciated and now should be used only to support older browsers (but you can still find it in older projects).



var waistSize = 80; // don't use unless you must
waistSize = 100; // can be re-assigned

...is a one-line code for a simple "if...else" statement and returns one of two expressions, just such:



condition ? expression_if_true : expression_if_false;


let retirementAge = 65;

if, else, else if

Control flow statements

"if" is the simpliest of all statements. Executes the code in it only if the condition (in the brackets) is true.
"else" has an extra block of code that executes if condition in the "if" statement is false.
"else...if" gives you even more different blocks of code that are assigned to different conditions.


if (asnwer === "yes") {
  return gameContinue();
}

if (age >= 18) {
  ageValidation = pass;
} else {
  ageValidation = fail;
}

if (year == 2025) {
  time = present;
} else if (year > 2025) {
  time = future;
} else if (year < 2025) {
  time = past;
} else {
  return error404();
}

Switch

another control flow statement

    Works as an alternative to "else...if" statement, but can execute code for more than one case. It performs different actions based on different options (cases). After executing the code for a certain case, it uses "break" keyword to exit from itself. If it's missing, the code will execute further.
    The last (optional) case is named "default" and is similar to "else" in "else...if" statement.

switch (expression) {
  case value1:
    return execution1;// Code to execute if expression === value1
    break;
  case value2:
    return execution2;// Code to execute if expression === value2
    break;
  ((case...break;) * N)// ... more cases
  default:
    // Code to execute if none of the cases match
}

switch (colourLike) {
  case "blue":
    console.log("blue is the sky");
    break;
  case "green":
    console.log("green is the grass");
    break;
  case "pink":
    console.log("pink is the pig");
    break;
  default:
    console.log("no colors for today, wait until tomorrow");
}

for, while, do...while

basic loops

The loop "for" we use when we know how many times we want to loop.
The "while" start the loop only when the condition is true, then keeps looping as long as it stays true.
The "do...while" is guaranteed to execute at least once. After first execusion it checks for the condition.


for (let i = 0; i < 3; i++) {
  console.log("The number is " + i);
}

let i = 0;
while (i < 3) {
  console.log("The number is " + i);
  i++; -->    -->    -->    -->    // remember to place condition here
} to avoid an infinite loop!

let i = 5;
do { // this guarantees execution
  console.log("The number is " + i);
  i++; -->    -->    -->    -->        // avoid an infinite loop!
} while (i < 3);


REMEMBER:
To prevent a "while" / "do...while" loop from becoming infinite, a statement inside the loop must modify the condition so it can eventually become false.

for...of, for...in

other useful loops

    for...of
- Loop iterates over the values of iterable objects (ie: arrays, strings, maps, sets).
    for...in
- This loop iterates over the enumerable properties of an object. It's not recommended for arrays.


const fruits = ['pear', 'kiwi', 'grspes', 'avocado', 'tomatoe'];
for (const fruit of fruits) {
console.log(fruit);
}

const person = { name: "Alice", age: 30 };
for (const key in person) {
console.log(`${key}: ${person[key]}`);
}
